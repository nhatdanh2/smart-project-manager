# Smart PM — Mobile (Expo)

React Native client built with **Expo + expo-router**.  Shares the
same FastAPI backend as the web app — the same JWT works in both
places (you can be logged in on web and mobile simultaneously).

## Stack

* **Expo SDK 51** with expo-router 3
* **Nativewind** (Tailwind for RN)
* **TanStack Query** for data fetching / cache
* **expo-secure-store** to persist tokens
* **expo-local-authentication** for biometric login (Face ID / Touch ID / fingerprint)
* **expo-notifications** for push (token registered with the
  backend; server-side `expo-server-sdk` delivers them)

## Running locally

```bash
cd mobile
npm install
# Start the dev server (QR code for Expo Go, or build for simulator)
npx expo start

# iOS simulator
npx expo start --ios

# Android emulator
npx expo start --android
```

By default the app points at `http://localhost:8000`.  For
physical devices, edit `app.json` → `expo.extra.apiBaseUrl` to
your LAN IP (e.g. `http://192.168.1.10:8000`).

## Biometric login

After the first successful password login the user is offered
"Enable biometric login" from the Profile tab.  This stores the
user profile (not the password) in SecureStore; the next launch
prompts Face ID / fingerprint, then refreshes the JWT against
the backend.

## Push notifications

On every login the app:

1. Asks for notification permission
2. Reads the Expo push token
3. `POST /api/push/tokens` on the backend

When the backend creates an in-app notification it also dispatches
a push to all of that user's devices via `expo-server-sdk`.  The
notification's `data.link` is what the app navigates to on tap.

## Building for production

```bash
# Configure EAS
npm install -g eas-cli
eas login
eas build:configure

# Internal build (store distribution = .apk / .ipa)
eas build --profile preview --platform all

# Submit to stores
eas submit --platform ios
eas submit --platform android
```

See `eas.json` (generated by `eas build:configure`) for the
production profile.

## Adding a new screen

1. Drop a file into `app/` following the expo-router convention.
2. If it requires auth, nest it under `app/(tabs)/` or another
   authenticated route group so the `AuthProvider` redirect works.
3. Use `useQuery` from `@tanstack/react-query` for data fetching.
4. The axios client in `src/lib/api.ts` already attaches the
   `Authorization` header.
